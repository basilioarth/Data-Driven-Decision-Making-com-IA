# Introdução
A aula aplica **técnicas avançadas de Prompt Engineering** em exemplos reais com **ChatGPT** (referido como "chat APT"). O objetivo é mostrar como **evoluir um prompt simples** para um **prompt estruturado, profissional e acionável**, usando **Systematic Role Framing**, **Context Enrichment** e **Constraint-Driven Prompts** — tudo alinhado à **anatomia do prompt**.

---

# Exemplo Prático: ChatGPT

## Prompt Simples (Básico)
> `"Compare Kafka e Python em termos de velocidade, facilidade de uso e casos comuns. Responda em tabela."`

**Limitações**:
- Sem papel definido → resposta genérica.
- Sem público-alvo → linguagem pode ser complexa.
- Sem restrição clara → pode extrapolar.

---

## Prompt Avançado (Profissional)

> ```
> Você é um **especialista em tecnologia com 10 anos de experiência**.  
> Compare **Kafka** e **Python** em termos de:  
> - Velocidade  
> - Facilidade de uso  
> - Casos comuns  
>  
> Responda em uma **tabela clara com 3 colunas**:  
> 1. Critério  
> 2. Kafka  
> 3. Python  
>  
> Use **linguagem simples, adequada para um público técnico iniciante**.  
> Inclua uma **breve conclusão no final**.
> ```

---

## Técnicas Aplicadas

| Técnica | Aplicação no Prompt |
|--------|---------------------|
| **Systematic Role Framing** | `"Especialista com 10 anos de experiência"` → ativa expertise realista. |
| **Context Enrichment** | `"Público técnico iniciante"` → define quem vai ler. |
| **Constraint-Driven Prompts** | `"Apenas 3 critérios"`, `"Tabela com 3 colunas"`, `"Linguagem simples"` → limites rígidos. |
| **Output as Artifact** | `"Tabela clara"` + `"Conclusão breve"` → entregável pronto para relatório. |

---

## Saída Esperada (Artefato Prático)

| **Critério** | **Kafka** | **Python** |
|-------------|----------|-----------|
| **Velocidade** | Alta em streaming (milhões de msg/s) | Boa em scripts, mas não para alta concorrência |
| **Facilidade de uso** | Configuração complexa, curva de aprendizado alta | Sintaxe simples, ideal para iniciantes |
| **Casos comuns** | Logs, eventos em tempo real, pipelines de dados | Automação, análise de dados, APIs leves |

**Conclusão**:  
> *Python é ideal para prototipagem rápida e análise. Kafka brilha em sistemas de eventos de alta escala. Use Python para começar; escale com Kafka quando necessário.*

---

## Benefícios do Prompt Avançado

| Vantagem | Impacto |
|--------|--------|
| **Evita extrapolação** | Não compara com RabbitMQ ou Java. |
| **Linguagem acessível** | Perfeito para estagiários ou gerentes técnicos. |
| **Entregável imediato** | Copie → cole no Notion, relatório ou apresentação. |
| **Sugestões controladas** | Pode sugerir alternativas, mas **só após cumprir o pedido**. |

# Conclusão
Este exemplo com **ChatGPT** prova:  
> **Um prompt bem estruturado = um consultor sênior em segundos.**

Você não precisa de todas as técnicas — mas **combinando Role Framing, Context Enrichment e Constraints**, você transforma uma pergunta vaga em um **relatório técnico claro, útil e profissional**.

**Técnicas usadas aqui**:
- **Role Framing** → especialista com experiência.
- **Context Enrichment** → público iniciante.
- **Constraints** → apenas 3 critérios, tabela, tom simples.
- **Output as Artifact** → tabela + conclusão.
