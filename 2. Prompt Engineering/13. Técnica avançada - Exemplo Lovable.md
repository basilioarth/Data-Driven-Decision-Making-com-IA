# Introdução
A aula aplica **técnicas avançadas de Prompt Engineering** no **Lovable**, a IA especializada em **prototipação e desenvolvimento de software**. O foco é **estruturar o prompt como uma receita de bolo** — com **etapas claras, ordem lógica e entregáveis concretos** — para gerar um **script Python completo, documentado e com saída em PNG**, ideal para automação de dados.

---

# Exemplo Prático: Lovable

## Prompt Simples (Básico)
> `"Crie um script em Python que leia um arquivo JSON com registros de vendas, filtre entradas acima de mil dólares e gere um gráfico em PNG."`

**Problemas**:
- Ordem confusa → IA pode gerar gráfico antes de ler o JSON.
- Sem documentação → código difícil de manter.
- Sem clareza no fluxo → risco de erro.

---

## Prompt Avançado (Profissional)

> ```
> Você é um **engenheiro Python focado em automação de dados**.  
>  
> Crie um **script em Python bem documentado** que:  
> 1. Leia o arquivo `vendas.json`  
> 2. Filtre registros com `valor > 1000`  
> 3. Gere um **gráfico em PNG** mostrando as vendas filtradas  
>  
> Use **comentários explicativos em cada etapa**.  
> Escreva o código de forma **clara e eficiente**.  
> Salve o gráfico como `vendas_acima_1000.png`.
> ```

---

## Técnicas Aplicadas

| Técnica | Aplicação no Prompt |
|--------|---------------------|
| **Systematic Role Framing** | `"Engenheiro Python focado em automação"` → perfil técnico especializado. |
| **Progressive Deepening** | Etapas numeradas (1. ler, 2. filtrar, 3. plotar) → fluxo lógico. |
| **Output as Artifact** | `"Salve como vendas_acima_1000.png"` → arquivo real gerado. |
| **Constraint-Driven Prompts** | `"Apenas PNG"`, `"Comentários em cada etapa"` → limites rígidos. |
| **Context Enrichment** | `"vendas.json"`, `"valor > 1000"` → contexto claro do dado. |


## Analogia da Receita de Bolo (da professora)

> **Fazer bolo ≠ abrir forno e tirar pronto.**  
> **Fazer script ≠ "gere um gráfico" sem passos.**

| Etapa do Bolo | Etapa do Script |
|--------------|-----------------|
| Pegar ingredientes | Ler JSON |
| Misturar massa | Filtrar dados |
| Aquecer forno | Configurar plot |
| Assar | Gerar PNG |

> **Ordem importa. Fluxo importa. Etapas importam.**

---

## Por que esse prompt funciona no Lovable?

| Vantagem | Impacto |
|--------|--------|
| **Etapas setorizadas** | Lovable separa em sessões → fácil alterar depois. |
| **Ordem lógica** | IA não gera gráfico antes de filtrar. |
| **Artefato real** | PNG + script → pronto para CI/CD ou dashboard. |
| **Manutenível** | Comentários → time entende e herda. |

# Conclusão
No **Lovable**, **Prompt Engineering é arquitetura de software**.  
Você não pede um gráfico — você **orquestra um pipeline completo**, com:
- **Engenheiro especializado**,
- **Etapas numeradas**,
- **Código documentado**,
- **Saída em PNG**.

**Técnicas usadas aqui**:
- **Role Framing** → engenheiro de automação.
- **Progressive Deepening** → 1. ler, 2. filtrar, 3. plotar.
- **Output as Artifact** → `vendas_acima_1000.png`.
- **Constraints** → PNG, comentários, eficiência.
