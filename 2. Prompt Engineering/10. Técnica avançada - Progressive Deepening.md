# Introdução
A aula apresenta a sexta técnica avançada de **Prompt Engineering**: **Progressive Deepening** (ou *Aprofundamento Progressivo*). É a estratégia de **não pedir tudo de uma vez**, mas avançar em **etapas claras**, refinando o resultado como em uma conversa estruturada — ideal para ferramentas como **Lovable**, onde sessões separadas permitem ajustes localizados sem quebrar o todo.

# Conteúdo Principal

## Técnica 6: **Progressive Deepening**  
*(Aprofundamento Progressivo – Visão Macro → Micro)*

### O que é?
- **Começa com visão geral** → **refina por etapas**.
- Cada prompt é um **follow-up** do anterior.
- Simula um **processo iterativo de design ou desenvolvimento**.

---

## Fluxo Típico (Exemplo Prático)

| Etapa | Prompt | Resultado |
|------|--------|---------|
| **1. Visão Geral** | `"Explique em alto nível a arquitetura de um dashboard de vendas com upload de CSV."` | Diagrama macro: frontend, backend, banco, IA. |
| **2. Refinamento** | `"Agora detalhe a parte de ingestão de dados: como ler CSV, validar e armazenar."` | Fluxo com pandas, validação de schema, S3. |
| **3. Código** | `"Gere o código Python para a etapa de ingestão, com tratamento de erros."` | Script completo com `try/except`, logging. |
| **4. Ajuste Local** | `"Mude a cor do gráfico de vendas para azul escuro e adicione filtro por região."` | Apenas o componente visual é alterado. |

---

## Por que usar Progressive Deepening?

| Benefício | Impacto Real |
|---------|-------------|
| **Evita caos** | Prompt gigante → resposta confusa. Etapas → clareza. |
| **Exploração interativa** | Como um **pair programming com IA**. |
| **Ajustes localizados** | Perfeito para **Lovable**: muda só a sessão de login, sem tocar no resto. |
| **Natural para devs** | Segue o **mesmo fluxo mental** de resolver bugs ou implementar features. |

---

## Onde brilha?

| Ferramenta | Uso Ideal |
|-----------|----------|
| **Lovable** | Prototipagem de apps/dashboards — altera cor, layout, lógica por sessão. |
| **Cursor AI** | Gera código por módulo (ingestão → processamento → API). |
| **ChatGPT / Claude** | Análise de dados em camadas: visão geral → query SQL → otimização. |

---

## Dica de Implementação

1. **Sempre referencie o anterior**:
   - `"Baseado na arquitetura que você descreveu, agora..."`  
2. **Mantenha sessões curtas e objetivas**:
   - 1 ideia por prompt.
3. **Use numeração ou marcadores**:
   - `"Etapa 1/5: ..."` → facilita o follow-up.

> **Regra de ouro**:  
> *“Um prompt grande = uma resposta bagunçada.  
> Um prompt por etapa = um artefato perfeito.”*

# Conclusão
**Progressive Deepening** é o **método de trabalho do futuro com IA**.  
Você não despeja um problema — você **constrói a solução junto com a IA**, etapa por etapa, como um arquiteto refinando um projeto.  
Perfeito para:
- **Lovable** (prototipagem modular),
- **Cursor** (código por camadas),
- **Análises complexas** (do macro ao SQL otimizado).

Combinado com:
- **Role Framing** → especialista,
- **Context Enrichment** → realidade,
- **Constraints** → limites,
- **Multilens** → múltiplas visões,
- **Output as Artifact** → entregáveis prontos,

...forma o **fluxo completo de engenharia com IA**:  
**visão → refinamento → código → ajuste → produto final**.  
Nas próximas técnicas: **Chain of Thought**, **Few-Shot** e **automação de pipelines inteiros** — tudo com **Progressive Deepening como base**.
