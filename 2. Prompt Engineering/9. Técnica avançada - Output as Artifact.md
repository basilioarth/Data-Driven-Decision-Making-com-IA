# Introdução
A aula apresenta a quinta técnica avançada de **Prompt Engineering**: **Output as Artifact** (ou *Entregáveis Concretos*). O foco é **forçar a IA a gerar saídas prontas para uso** — como checklists, JSONs, tabelas, scripts ou arquivos — eliminando retrabalho e transformando a resposta em um **artefato operacional** imediato.

# Conteúdo Principal

## Técnica 5: **Output as Artifact**  
*(Saída como Produto Prático)*

### O que é?
- A IA **não apenas responde** — ela **entrega um artefato útil**.
- O output é **estruturado, formatado e acionável** desde o primeiro momento.
- Relacionado ao **formato de saída** da anatomia do prompt, mas com foco em **usabilidade prática**.

---

## Exemplos de Artefatos Concretos

| Prompt | Artefato Gerado |
|--------|------------------|
| `"Crie um checklist de deploy para uma API em Node.js."` | Lista com 10 passos em Markdown, pronta para colar no Notion. |
| `"Gere uma tabela comparativa entre AWS Lambda, GCP Cloud Functions e Azure Functions."` | Tabela Markdown com colunas: Preço, Cold Start, Limites, etc. |
| `"Retorne um JSON com parâmetros de configuração para um job Spark."` | ```json { "spark.executor.memory": "4g", "spark.sql.shuffle.partitions": 200 } ``` |
| `"Escreva um script SQL para limpar CPFs inválidos."` | Script `.sql` executável com `WHERE LENGTH(cpf) != 11 OR ...` |

---

## Por que usar Output as Artifact?

| Benefício | Impacto Real |
|---------|-------------|
| **Economia de tempo** | Nada de copiar, formatar ou limpar depois. |
| **Zero retrabalho** | Saída já é **input para outra ferramenta** (Notion, Git, CI/CD, API). |
| **Padronização** | Todos os artefatos seguem o mesmo formato. |
| **Integração direta** | JSON → backend, SQL → banco, Markdown → documentação. |

---

## Dica Prática: Como garantir um bom artefato?

1. **Seja explícito no formato**:
   - `"Retorne APENAS JSON válido, sem texto adicional."`
   - `"Use bullet points com emojis de check (✅)."?`
2. **Valide a saída**:
   - Se o JSON estiver quebrado → peça correção.
   - Se o SQL não rodar → teste e itere.
3. **Escolha a IA certa**:
   - **Cursor AI** → scripts e JSONs perfeitos.
   - **Manus AI** → tabelas e relatórios visuais.
   - **Claude** → documentos longos e estruturados.

> **Regra de ouro**:  
> *“Peça o que você vai colar. Não peça o que você vai reescrever.”*
