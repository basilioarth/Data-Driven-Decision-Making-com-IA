# Resumo da Aula – Segunda e Terceira Tentativas com Lovable AI

## Problema confirmado nas duas primeiras tentativas
- Lovable **não é boa em fazer a análise estatística do zero**  
- Ela alucina números (300 clientes → 10 ou 80, churn 32 % → 26,7 % ou 30 %)  
- Erra contagens básicas e proporções  
- Os gráficos ficam lindos, mas os dados dentro estão errados → não dá para confiar

## Conclusão clara do professor
| Tarefa                          | Melhor ferramenta          |
|---------------------------------|----------------------------|
| Análise estatística profunda    | **Manus AI** (ou Python)   |
| Dashboard bonito e interativo   | **Lovable** (mas só com dados já validados) |

**Fluxo correto (o que realmente funciona na prática):**
1. Fazer toda a análise pesada com **Manus AI** → relatório 100 % confiável  
2. Copiar os números exatos + insights prontos  
3. Colar na Lovable com um prompt do tipo:  
   > “Crie um dashboard interativo lindo usando exatamente estes dados e insights que eu te passo abaixo. Não faça nenhuma análise nova, só visualize bonitinho.”

## Terceira tentativa (em andamento)
Professor vai testar exatamente esse fluxo:
- Prompt ultra-simples + números já validados da Manus AI  
- Só pedir design e interatividade (filtros, hover, cores bonitas, etc.)

Assim o dashboard fica:
- Visualmente perfeito  
- 100 % correto nos números  
- Compartilhável com o time todo em 1 clique

Próximo resumo: resultado final do dashboard com o prompt corrigido → vai ficar lindo e confiável!
