# Introdução
A aula apresenta o **ciclo de tomada de decisão com dados** (DDDM), detalhando os quatro passos fundamentais: coleta, análise, interpretação e decisão. O foco é ensinar como transformar dados brutos em insights acionáveis, evitando vieses e promovendo decisões estratégicas com base em evidências concretas.

# Conteúdo Principal

## 1. Coleta
- **Origem dos dados**: Identificar de onde vêm as informações.
  - Dados gerados pelo próprio trabalho?
  - Banco de dados existente?
  - Outras fontes estruturadas?
- Essencial para garantir que os dados sejam relevantes e confiáveis desde o início.

## 2. Análise (Processamento + Limpeza + Modelagem)
- **Limpeza**: Remover dados incompletos, com erros, duplicados ou em branco.
- **Modelagem**: Estruturar os dados para análise eficiente.
- **Definição de perguntas**: Durante a limpeza, já surge a reflexão:
  - *Quais perguntas quero responder com esses dados?*
  - Exemplo: Em dados de uso de app, a coluna "região" é relevante? Pode indicar problemas regionais?
- **Seleção consciente**: Descartar o que não contribui para a análise.

## 3. Interpretação
- Buscar **respostas reais** às perguntas definidas.
- Aceitar que os dados podem **contradizer expectativas** — isso é o objetivo do DDDM: evitar enviesamento.
- **Storytelling dos dados**: Entender a *história* que os números contam.
  - Exemplo: “A lampadazinha acendeu” → insight claro sobre um problema ou oportunidade.

## 4. Decisão
- A decisão é **multivariada** e baseada na interpretação:
  - **Pivotar** (mudar direção do produto, processo ou ferramenta).
  - **Implementar algo novo**.
  - **Manter o curso** (se os dados mostram resultado positivo ou neutro).
  - **Não fazer nada** (quando não há necessidade de mudança).
- O dado orienta a **ação estratégica**, não apenas confirma hipóteses.

## Visão geral do ciclo
1. **Coleta** → De onde vêm os dados?
2. **Análise** → Limpeza + modelagem + definição de perguntas.
3. **Interpretação** → Quais respostas os dados trazem? Qual é a história?
4. **Decisão** → Qual ação (ou inação) tomar com base nisso?

> *“O processo parece simples, mas quando você olha pro dashboard e ele responde exatamente à sua pergunta, é muito bacana.”*

# Conclusão
O ciclo de DDDM é um fluxo estruturado e iterativo que transforma dados brutos em decisões estratégicas. Começa com a coleta intencional, passa por uma análise crítica guiada por perguntas claras, evolui para a interpretação objetiva (sem vieses) e culmina em ações (ou inações) fundamentadas. Dominar esses quatro passos permite a qualquer profissional — especialmente desenvolvedores — extrair valor real dos dados, gerar impacto mensurável e orientar o negócio com precisão e confiança.
