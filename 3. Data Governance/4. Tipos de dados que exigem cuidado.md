# Tipos de Dados CRÍTICOS (LGPD, Reputação, Ética)

> **Todo dado é importante. Mas alguns são INTOCÁVEIS.**

---

## **1. PII – Personally Identifiable Information**  
*(Dados que **identificam uma pessoa física**)*

| Exemplo | Risco |
|--------|------|
| CPF, RG, Passaporte | Roubo de identidade |
| Nome completo + data de nascimento | Fraude bancária |
| Endereço residencial | Invasão, stalking |
| E-mail + telefone | Phishing direcionado |

> **Regra**: **NUNCA sai da empresa sem anonimização.**

---

## **2. Dados Financeiros**  
*(Qualquer coisa que movimente ou identifique dinheiro)*

| Exemplo | Risco |
|--------|------|
| Número do cartão, CVV, validade | Fraude imediata |
| Chave PIX (e-mail/celular) | Transferência indevida |
| Extrato bancário, saldo | Chantagem, extorsão |
| Histórico de transações | Perfil de consumo exposto |

> **Regra**: **Processar apenas em ambiente seguro (on-premise ou com DPA).**

---

## **3. Segredos Comerciais & Informações Privilegiadas**  
*(O "molho secreto" da empresa)*

| Exemplo | Risco |
|--------|------|
| Fórmula do refrigerante | Cópia por concorrente |
| Estratégia de precificação | Perda de mercado |
| Plano de expansão (antes do anúncio) | Insider trading |
| Código-fonte proprietário | Roubo de IP |

> **Exemplo real**:  
> *"Como escalar o refrigerante preto e docinho pra 1M de pessoas?"* → **vazou a receita**.

> **Regra**: **NUNCA discuta com IA externa. Nem com amigo de fora.**

---

## **4. Dados Sensíveis Especiais (LGPD Art. 5º, II)**  
*(Dados com **alto risco de discriminação**)*

| Tipo | Exemplo | Risco |
|-----|--------|------|
| **Saúde** | Tipo sanguíneo, HIV, câncer | Discriminação em seguro/emprego |
| **Biometria** | Impressão digital, facial | Falsificação de identidade |
| **Opinião Política** | Filiado a partido X | Perseguição política |
| **Filiação Sindical** | Membro do sindicato Y | Represália trabalhista |
| **Orientação Sexual** | LGBTQIA+ | Discriminação social |
| **Origem Étnica** | Indígena, afrodescendente | Racismo institucional |

> **Exemplo grave**:  
> Lista de doadores de órgãos vazada → **vidas em risco**.

> **Regra**: **Armazenamento criptografado + acesso restrito + NUNCA externo.**

## **Checklist: Este dado pode ir pra IA externa?**

- [ ] É **PII** (nome, CPF, e-mail)? → **NÃO**  
- [ ] É **financeiro** (cartão, PIX)? → **NÃO**  
- [ ] É **segredo comercial** (estratégia, IP)? → **NÃO**  
- [ ] É **sensível especial** (saúde, política)? → **NÃO**  

> **Se marcar qualquer "sim" → ANONIMIZE ou USE IA LOCAL.**

---

## **Como Proteger? (3 Ações Imediatas)**

```python
# 1. ANONIMIZE (sempre)
df['cliente_id'] = range(1, len(df)+1)
df = df.drop(columns=['nome', 'cpf', 'email', 'cartao', 'endereco'])

# 2. VALIDE
assert not any(col in df.columns for col in ['cpf', 'nome', 'email'])

# 3. USE IA LOCAL para dados reais
# → Llama 3, Mistral, ou Azure OpenAI com Private Endpoint
```
