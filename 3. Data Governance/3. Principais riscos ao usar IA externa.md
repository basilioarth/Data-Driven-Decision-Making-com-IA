# Riscos Críticos ao Usar IA Externa (ChatGPT, Claude, Manus, Lovable, etc.)

> **Você não controla o data center. Você controla o dado.**  
> **Um vazamento pequeno pode virar um desastre global.**

---

## **4 Riscos Principais (Efeito Dominó)**

| Risco | Descrição | Impacto |
|------|----------|--------|
| **1. Vazamento de Dados Sensíveis** | Envio de **PII** (nome, CPF, e-mail, comportamento) para servidores externos | **Exposição imediata** |
| **2. Violação de Compliance** | Infringir **LGPD, GDPR, HIPAA** ao vazar dados sem consentimento | **Multas até 2% do faturamento** |
| **3. Risco Reputacional** | Vazamento vira notícia → clientes perdem confiança | **Churn, queda de ações, fechamento** |
| **4. Dependência de Terceiros** | Você **não controla** como a IA armazena ou usa seus dados | **Sem poder de remediação** |

---

## **Exemplo Real (e Comum)**

> **Planilha com 5.000 acessos de usuários (mês/região)** enviada ao ChatGPT:

```csv
usuario_id,nome,email,regiao,acessos_mes
1,Ana Silva,ana@email.com,Sudeste,42
2,João Santos,joao@email.com,Nordeste,19
```

**O que aconteceu?**  
- **5.000 CPFs/e-mails expostos**  
- **Sem anonimização**  
- **OpenAI agora tem seus dados**  
- **Você não sabe onde estão, por quanto tempo, nem quem acessa**

## **Efeito Dominó: De 1 Planilha a 1 Crise**

```
[1] Você envia planilha → IA externa

## **Dependência de Terceiros: O Pior Risco**

| Você | Provedor (OpenAI, etc.) |
|-----|-------------------------|
| Quer **controle total** | Tem **missão própria** |
| Quer **remediação imediata** | Responde em **semanas** |
| Quer **excluir dados** | Pode **manter para treino** |
| Quer **transparência** | Dá **resposta genérica** |

> **Você é o "terceiro" para eles.**  
> **Seus dados = combustível do modelo.**

## **Como Proteger? (3 Regras de Ouro)**

```markdown
### Regra 1: **NUNCA envie PII para IA externa**
- CPF, nome, e-mail, telefone, endereço → **NUNCA**

### Regra 2: **Anonimize SEMPRE**
```python
df['user_id'] = range(1, len(df)+1)
df = df.drop(columns=['nome', 'email', 'cpf', 'telefone'])
```

### Regra 3: **Use IA local para dados reais**
| Dados | Onde rodar |
|------|-----------|
| **Reais / Sensíveis** | Llama 3, Mistral **no seu servidor** |
| **Fictícios / Anonimizados** | ChatGPT, Claude, Manus |


## **Checklist ANTI-VAZAMENTO (5 Perguntas)**

- [ ] Tem **nome, CPF, e-mail** no dado?  
- [ ] Estou enviando para **IA externa**?  
- [ ] Já **anonimizei 100%**?  
- [ ] Posso rodar isso **localmente**?  
- [ ] O cliente **sabe que seus dados vão pra nuvem**?

> **Se 1 resposta for "sim" sem anonimização → NÃO ENVIE.**


## **Prompt Padrão de Segurança (Use Sempre)**


> ANTES DE QUALQUER ANÁLISE:
> 1. **Anonimize todos os dados pessoais** (nome, CPF, e-mail, telefone, endereço).
> 2. Substitua por **IDs numéricos**.
> 3. **Confirme por escrito** que não há PII.
> 4. Só então prossiga com a análise.


## **Alternativas Seguras**
