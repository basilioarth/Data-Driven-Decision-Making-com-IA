# RESUMO COMPLETO DA PLANILHA DE CHURN (Dados 100% Sintéticos – Prontos para IA)

**Arquivo**: `churn_plataforma_sintetico.csv`  
**Linhas**: ~1.000 clientes (ativo + churn)  
**Objetivo**: Descobrir os **grandes motivadores de cancelamento** e sugerir ações imediatas

### Colunas da Planilha (Traduzidas + Explicadas)

| Coluna                | Nome Original         | Tipo          | O que significa (em linguagem de dev/produto) |
|-----------------------|-----------------------|---------------|-----------------------------------------------|
| CustomerID            | CustomerID            | int           | ID fictício (já anonimizado)                  |
| Age                   | Age                   | int           | Idade do cliente                              |
| Gender                | Gender                | M/F/Other     | Gênero (inclui “Other”)                       |
| Region                | Region                | texto         | Sudeste, Nordeste, etc.                       |
| PlanType              | PlanType              | Basic / Premium | Tipo de plano                                 |
| MonthlyFee            | MonthlyFee            | float         | Valor mensal (49,90 ou 99,90)                 |
| SatisfactionScore     | SatisfactionScore     | 1–5           | Nota de satisfação (NPS-like)                 |
| SupportTickets        | SupportTickets        | int           | Quantidade de tickets abertos                 |
| PlatformLogins        | PlatformLogins        | int           | Quantidade de acessos no último mês           |
| Churn                 | Churn                 | 0 / 1         | 1 = cancelou, 0 = ativo                       |
| ChurnReason           | ChurnReason           | texto         | Motivo declarado no cancelamento             |

### Motivos de Churn já mapeados (ChurnReason)

| Motivo               | Significado real para o time                                                                 |
|----------------------|-----------------------------------------------------------------------------------------------|
| Cost                 | “Tá caro” – reclamação direta de preço                                                       |
| Competition          | Cliente foi para concorrente (provavelmente mais barato ou com mais features)                 |
| Accessibility        | Problema de acessibilidade (deficiência visual, leitor de tela, etc.)                         |
| Bug                  | Bug crítico que travou o uso                                                                  |
| Difficulty of Use    | UX/UI ruim – “não entendo como usar”                                                          |
| None                 | Cliente ativo (Churn = 0)                                                                     |
| Other                | Motivo imprevisível (iPad quebrou, mudança de país, etc.) – normalmente < 2% dos casos        |

### Por que isso importa para DEVS?

| Motivo            | Ação direta do time de desenvolvimento                                      |
|-------------------|------------------------------------------------------------------------------|
| **Bug**           | → Abrir ticket urgente + hotfix                                              |
| **Difficulty of Use** | → Revisar fluxo crítico + melhorar onboarding / tooltips                 |
| **Accessibility** | → Implementar WCAG 2.1 (leitor de tela, contraste, navegação por teclado)   |
| **Cost / Competition** | → Sugerir novo plano intermediário ou feature que justifique o valor     |
